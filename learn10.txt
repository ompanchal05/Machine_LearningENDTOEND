CASE STUDY 1: HOUSE PRICE PREDICTION (REGRESSION)
=================================================

Problem Statement:
------------------
Predict house prices based on features like:
- Area
- Location
- Number of rooms
- Age of house

Business Goal:
--------------
Help buyers and sellers estimate fair house prices.

-------------------------------------------------

Step 1: Data Understanding
--------------------------
- Dataset contains numerical and categorical features
- Target variable: House price (continuous)

-------------------------------------------------

Step 2: Data Cleaning
---------------------
- Handle missing values (median for numeric)
- Encode categorical features
- Remove obvious outliers

-------------------------------------------------

Step 3: Feature Engineering
---------------------------
- Price per square foot
- Age of house bucketed
- Location encoding

-------------------------------------------------

Step 4: Model Selection
-----------------------
- Start with Linear Regression (baseline)
- Try Ridge and Lasso to reduce overfitting
- Compare using RMSE

-------------------------------------------------

Step 5: Evaluation
------------------
- Metric used: RMSE
- Cross-validation for stability

-------------------------------------------------

Step 6: Final Outcome
---------------------
- Ridge Regression performed best
- Model generalized well on unseen data

-------------------------------------------------

Interview Explanation Line:
---------------------------
"I built a regression model to predict house prices, starting with a baseline linear model, improving it using regularization, and evaluating performance using RMSE and cross-validation."

CASE STUDY 2: CUSTOMER CHURN PREDICTION (CLASSIFICATION)
=================================================

Problem Statement:
------------------
Predict whether a customer will leave the service.

Business Goal:
--------------
Reduce customer loss by early identification.

-------------------------------------------------

Step 1: Data Understanding
--------------------------
- Binary target: Churn (Yes/No)
- Imbalanced dataset

-------------------------------------------------

Step 2: Data Cleaning
---------------------
- Handle missing values
- Encode categorical variables
- Scale numerical features

-------------------------------------------------

Step 3: Feature Engineering
---------------------------
- Tenure groups
- Usage frequency
- Payment behavior features

-------------------------------------------------

Step 4: Model Selection
-----------------------
- Logistic Regression (baseline)
- KNN (after scaling)
- Random Forest for non-linearity

-------------------------------------------------

Step 5: Evaluation
------------------
- Metrics: Precision, Recall, F1-score
- Recall prioritized (catch churners)

-------------------------------------------------

Step 6: Final Outcome
---------------------
- Random Forest gave best recall
- Used class weighting to handle imbalance

-------------------------------------------------

Interview Explanation Line:
---------------------------
"I worked on a churn prediction problem where recall was critical, compared multiple classifiers, and selected a Random Forest model to better capture non-linear patterns."

=================================================
CASE STUDY 3: FRAUD DETECTION (IMBALANCED DATA)
=================================================

Problem Statement:
------------------
Detect fraudulent transactions.

Business Goal:
--------------
Prevent financial loss.

-------------------------------------------------

Key Challenges:
---------------
- Highly imbalanced data
- False negatives are costly

-------------------------------------------------

Approach:
---------
- Used Logistic Regression and Random Forest
- Applied class weights
- Used Precision–Recall instead of accuracy

-------------------------------------------------

Final Result:
-------------
- Improved recall significantly
- Reduced false negatives

-------------------------------------------------

Interview Explanation Line:
---------------------------
"In fraud detection, I focused on recall and precision instead of accuracy due to class imbalance, and tuned models accordingly."
