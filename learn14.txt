HOW TO ANSWER ANY ML SYSTEM DESIGN QUESTION
=================================================

GENERAL STRUCTURE:
------------------
1. Clarify business problem
2. Define inputs & outputs
3. Define success metrics
4. Design data pipeline
5. Feature engineering
6. Model choice
7. Training pipeline
8. Deployment
9. Monitoring & retraining
10. Trade-offs

Always speak in this order.

CASE STUDY 1: CUSTOMER CHURN PREDICTION SYSTEM
=================================================

Problem:
--------
Predict whether a customer will leave the service.

Business Goal:
--------------
Reduce customer churn by early intervention.

-------------------------------------------------

Inputs:
-------
- User activity
- Subscription history
- Usage frequency
- Support tickets

Output:
-------
- Churn probability (0 to 1)

-------------------------------------------------

Data Pipeline:
--------------
- Collect data from databases and logs
- Clean missing and incorrect values
- Aggregate user-level features

-------------------------------------------------

Feature Engineering:
--------------------
- Tenure (days since signup)
- Average usage per week
- Payment delay count
- Complaint frequency

-------------------------------------------------

Model Choice:
-------------
- Start with Logistic Regression (baseline)
- Try Random Forest / XGBoost for non-linearity

-------------------------------------------------

Training:
---------
- Train weekly using latest data
- Use cross-validation

-------------------------------------------------

Deployment:
-----------
- REST API for real-time prediction
- Batch prediction for daily reports

-------------------------------------------------

Monitoring:
-----------
- Track prediction distribution
- Track recall on churners
- Monitor data drift

-------------------------------------------------

Retraining:
-----------
- Retrain monthly
- Retrain immediately if performance drops

-------------------------------------------------

Trade-offs:
-----------
- Accuracy vs interpretability
- Latency vs complexity

-------------------------------------------------

One-Line Interview Answer:
--------------------------
"I designed a churn prediction system with data pipelines, feature engineering, model training, deployment, and continuous monitoring."

CASE STUDY 2: FRAUD DETECTION SYSTEM
=================================================

Problem:
--------
Detect fraudulent transactions in real time.

Business Goal:
--------------
Prevent financial loss.

-------------------------------------------------

Key Challenges:
---------------
- Highly imbalanced data
- Low latency requirement

-------------------------------------------------

Inputs:
-------
- Transaction amount
- Location
- Device info
- User history

Output:
-------
- Fraud score

-------------------------------------------------

Model Choice:
-------------
- Logistic Regression (fast baseline)
- Gradient Boosting for better accuracy

-------------------------------------------------

Evaluation Metrics:
-------------------
- Recall (most important)
- Precision
- False negatives

-------------------------------------------------

Deployment:
-----------
- Real-time API (<100ms latency)

-------------------------------------------------

Monitoring:
-----------
- Monitor false negatives
- Alert system for spikes

-------------------------------------------------

Trade-offs:
-----------
- Speed vs accuracy
- False positives vs false negatives

-------------------------------------------------

One-Line Interview Answer:
--------------------------
"I built a fraud detection system optimized for recall and low latency using a layered ML approach."

COMMON SYSTEM DESIGN QUESTIONS (INTERVIEW)
=================================================

Q1. How do you handle data drift?
A:
Monitor distributions and retrain models.

-------------------------------------------------

Q2. How do you scale ML systems?
A:
Use batch processing, caching, and distributed systems.

-------------------------------------------------

Q3. How do you ensure reliability?
A:
Fallback models, monitoring, alerts.
