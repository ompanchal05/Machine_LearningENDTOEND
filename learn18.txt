REAL MOCK INTERVIEWS — DATA → ML → ENGINEERING
=============================================

PURPOSE
-------
This document simulates real interview conversations.
Use it to:
- Practice speaking answers
- Build confidence
- Think clearly under pressure
- Prepare end-to-end for interviews

How to use:
-----------
- Read interviewer question aloud
- Answer in your own words
- Compare with sample answer
- Improve clarity (not memorization)

=================================================
MOCK INTERVIEW 1: DATA ANALYST
=================================================

Interviewer:
------------
Tell me about yourself.

Candidate (Sample Answer):
--------------------------
"I have a strong foundation in data analysis using SQL and Python. I work with datasets to clean data, perform exploratory analysis, and create insights that help business decisions."

-------------------------------------------------

Interviewer:
------------
How do you approach a new dataset?

Candidate:
----------
"I first understand the business goal, then check data size, columns, missing values, and data types. After that, I perform basic EDA to understand patterns and outliers."

-------------------------------------------------

Interviewer:
------------
Which SQL queries do you use most?

Candidate:
----------
"I mostly use SELECT, WHERE, GROUP BY, JOINs, and aggregate functions like COUNT, SUM, and AVG."

-------------------------------------------------

Interviewer:
------------
What is an example of insight you found from data?

Candidate:
----------
"I identified a drop in customer engagement during weekends, which helped the team adjust marketing timing."

=================================================
MOCK INTERVIEW 2: DATA SCIENTIST
=================================================

Interviewer:
------------
Explain one ML project you worked on.

Candidate:
----------
"I worked on a customer churn prediction project. I cleaned the data, engineered features, tried multiple models like Logistic Regression and Random Forest, evaluated them using recall and F1-score, and selected the best model."

-------------------------------------------------

Interviewer:
------------
Why did you choose Random Forest?

Candidate:
----------
"Because it handled non-linear patterns well and reduced overfitting compared to simpler models."

-------------------------------------------------

Interviewer:
------------
How do you handle overfitting?

Candidate:
----------
"I use cross-validation, regularization, simpler models, and ensure good feature engineering."

-------------------------------------------------

Interviewer:
------------
How do you evaluate a classification model?

Candidate:
----------
"I look beyond accuracy and focus on precision, recall, F1-score, and confusion matrix, depending on business cost."

=================================================
MOCK INTERVIEW 3: ML ENGINEER
=================================================

Interviewer:
------------
How do you deploy an ML model?

Candidate:
----------
"I save the trained model as a .pkl file and deploy it via an API or web app. I also ensure preprocessing is included using pipelines."

-------------------------------------------------

Interviewer:
------------
What is data leakage?

Candidate:
----------
"Data leakage happens when test data information is used during training, leading to unrealistically high performance."

-------------------------------------------------

Interviewer:
------------
How do you monitor models in production?

Candidate:
----------
"I monitor prediction distribution, input data drift, performance metrics, and latency."

-------------------------------------------------

Interviewer:
------------
What happens when model performance drops?

Candidate:
----------
"I investigate data drift and retrain the model using recent data."

=================================================
MOCK INTERVIEW 4: SYSTEM DESIGN (ML)
=================================================

Interviewer:
------------
Design a churn prediction system.

Candidate:
----------
"I would start by understanding business goals, collect user activity data, engineer meaningful features, train baseline and advanced models, deploy via API, and continuously monitor and retrain."

-------------------------------------------------

Interviewer:
------------
What trade-offs would you consider?

Candidate:
----------
"I would balance accuracy vs interpretability and latency vs model complexity."

=================================================
MOCK INTERVIEW 5: DEEP LEARNING
=================================================

Interviewer:
------------
When would you use deep learning?

Candidate:
----------
"When data is large and unstructured, like images or text, and simpler ML models are not sufficient."

-------------------------------------------------

Interviewer:
------------
Why Transformers are popular?

Candidate:
----------
"Because they handle long-range dependencies efficiently and allow parallel training using attention mechanisms."

------------
Tell me about a challenge you faced.

Candidate:

Interviewer:
